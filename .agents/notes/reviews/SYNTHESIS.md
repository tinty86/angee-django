## Tier 1 ‚Äî Critical / verified, act first

### C1. The `base` addon's migration escapes the composer pipeline  [S‚úìcrit, C‚úìhigh] ‚Äî VERIFIED
`Resource` is `abstract=True` (`resources/models.py:52`); the composer emits the
concrete model into `runtime/base/models.py`. But `_addon_config_classes` excludes
`BaseConfig` (`settings.py:111`), so `_migration_modules` never redirects `base`,
yet `base` IS in `plan.labels` ‚Üí `makemigrations` writes into the **installed
package source tree** `src/angee/base/migrations/0001_initial.py` with a wall-clock
header `# Generated by Django 6.0.5 on 2026-05-30 18:40`, never normalized.
Violates "edit source not artifact", byte-determinism, "no wall-clock in artifacts",
and "the framework core is an addon like any other". **VERIFIED firsthand.**
Fix: route `base` through the same runtime-migration redirect + normalization as
every other addon, then delete the committed source migration. (Codex frames the
deeper choice: either base is composed like everything else, or `Resource` becomes
a plain concrete base-app model and `runtime/base` is not emitted ‚Äî one owner.)

### C2. Tree is not green against the repo's own checks  [S‚úì] ‚Äî VERIFIED + broader
`uv run ruff check src/angee/base/` ‚Üí **5 errors** (all in the stray migration).
`ruff format --check` ‚Üí **7 files** would reformat: `emission.py`, `crud.py`,
`subscriptions.py`, the migration, `entries.py`, and 2 test files (subagent only
spotted crud.py). `docs/backend/guidelines.md` requires these to pass before handoff.

## Tier 2 ‚Äî High confidence (3‚Äì4 reviewers agree)

### H1. Process-global mutable `_active_ledger_model`  [M, S, G, C ‚Äî all four]
`widgets.py:16` module global, set/cleared via `global ‚Ä¶ # noqa` in a manual
try/finally in `managers.py:46,91`. Hidden, non-reentrant cross-module state.
Owner is the `AngeeResource` instance (it already carries `entry`). Fix: thread the
ledger model through the resource/widget factory; delete the global.

### H2. `sys.argv` build sniff  [M, S, G(crit)]
`apps.py:36` `_running_angee_build()` ‚Üí `sys.argv[1:3]==["angee","build"]`, read in
`import_models()` (:350). Fragile under `manage.py angee build`, `call_command`,
tests, `pipeline.run(addons=...)`. Owner of "is a build running" is the build
pipeline. Fix: explicit process-local/build-owned flag that `import_models()` reads.

### H3. `public_id` / `from_public_id` resolution scattered  [M, S, G]
Same "resolve instance by public id" reimplemented with `getattr(model,
'from_public_id',None)` + `isinstance(AngeeModel)` + `_default_manager.filter(pk=)`
fallbacks in `widgets._instance_from_public_id` (:152), `widgets.public_id` (:164),
`subscriptions._public_id` (:288), `crud._resolve_for_delete` (:132). Owner is
`AngeeModel`. The fallback is dead defensiveness ‚Äî every composed model IS an
AngeeModel. Fix: make `public_id`/`from_public_id` a typed (`Self | None`, not
`Any`) part of the composed-model contract; call directly.

### H4. `compose_defaults` mutates `sys.path` while documented "pure"  [S(low), C(high), M agree]
`settings.py:41` inserts `runtime_dir.parent` into `sys.path` then returns the
mapping ‚Äî its own docstring (:38) claims "pure function ‚Ä¶ does not read the
environment", and the guideline says settings helpers are pure. Fix: return the
path for the host to insert in one visible step, or a separately named side-effect
function the host calls explicitly.

## Tier 3 ‚Äî Real, single strong source (verified where checkable)

### M1. GraphQL "schema parts" is a parallel DSL vs the documented contract  [C] ‚Äî VERIFIED
`apps.get_schema_parts` + `schema.py` expect `graphql.py` to export
`schemas = {name: {query/mutation/subscription/types/extensions: [...]}}` (confirmed
in `examples/notes-angee/.../graphql.py:85`). But `docs/backend/guidelines.md:65`
and `docs/glossary.md:74` say addons "expose Strawberry `Schema` objects ‚Ä¶ does not
introduce a parallel schema language." This is a genuine code‚Üîdocs mismatch (AGENTS.md:
a bug requiring reconciliation). **NUANCE / architect call:** the parts layer exists
because multiple addons merge into one named schema (`merge_types`); pre-built Schema
objects per addon couldn't be merged the same way ‚Äî so the *code* design may be
correct and the *docs* stale. Reconcile deliberately; don't blindly "fix" the code.

### M2. `changes()` registers Django signals at schema-build time  [C, M noted]
`subscriptions.py:53` `connect_publishers(model)` runs inside the `changes()` surface
factory ‚Üí signal wiring depends on whether/when the schema is built. Violates
"compose at build time / don't register at runtime" (AGENTS.md). Fix: make `changes()`
pure declaration; register publishers from `AppConfig.ready()` / a build-owned registry.

### M3. 1:1 wrapper ceremony (two places)  [M]
(a) `BaseAddonConfig` pairs ~7 public `get_X()` with private `_X` cached_property
(`apps.py:95‚Äì337`) ‚Äî cached_property IS the Django-native public accessor; the getter
layer is pure ceremony. (b) `emission.py:38‚Äì117` has 8 public funcs that only unpack
`RuntimePlan` and forward to a `_private`; `RuntimePlan` owns the data ‚Üí these are
methods on the plan. AGENTS.md: "do not fix hidden magic by adding ceremony."

### M4. Dual-shape `getattr`+callable dispatch  [M, S]
`emission._declared_composition_fields` (:381) and `_model_reference` (:409) probe
`getattr(x,'method',None); if callable‚Ä¶else fallback` over (AngeeModel | plain mixin).
Comments call it "intentional polymorphism" ‚Äî which is the smell naming itself. Fix:
give extension bases a shared mixin so one type answers; or standardize references on
`_meta.label`/`label_lower` (Django-native dotted ref). [overlaps subagent #4]

### M5. `app_label.Model` label parsing reimplemented 3√ó  [S]
`mixins/models.py:55`, `managers.py:210`, `widgets.py:145` each call
`make_model_tuple` + resolve, with inconsistent error handling (`widgets._model_from_label`
leaks a bare `LookupError`). Fix: one `resolve_model(label)` primitive.

### M6. `_surface_name` + Strawberry introspection duplicated  [S]
`crud.py:187` and `schema.py:135` are byte-identical; both reach into Strawberry
private dunders. Fix: one `graphql/` introspection helper.

### M7. Extension collision check only covers fields  [C]
`emission._check_field_collisions` (:348) fails fast on declared fields only;
extension bases can still silently collide on managers/methods/properties/`Meta` ‚Äî
MRO picks a winner. AGENTS.md wants extension fail-fast. Fix: constrain extensions to
field-only mixins, or check all exported members.

### M8. URL-backed resources cached with no content pin  [C]
`fetch.py:31` caches by URL hash forever, no checksum/timeout ‚Äî source of truth is a
mutable remote, against deterministic load. Fix: require declared SHA-256 (+timeout),
cache by verified content, surface digest in review artifacts; or drop URL resources.

## Recommended refactor order (dependency-aware)
1. **C2 + C1**: make the tree green and fix the base-migration ownership (ruff is trivial;
   the migration is the headline correctness bug). Decide base's ownership (compose-like-others
   vs plain concrete base app) ‚Äî this is an architect decision that gates emission changes.
2. **H3 + M5 + M4**: one pass ‚Äî push `public_id`/`from_public_id`, label resolution, and
   model-reference formatting onto `AngeeModel`/`_meta`. Kills the biggest DRY/ownership cluster.
3. **H1 + H4 + H2 + M2**: remove hidden state/side-effects ‚Äî ledger global, `sys.path` mutation,
   argv sniff, runtime signal registration. All "explicit owner instead of ambient state".
4. **M3**: collapse the 1:1 wrapper ceremony (cached_property accessors; RuntimePlan methods).
5. **M1**: reconcile the GraphQL parts contract with the docs (architect decision first).
6. **M6, M7, M8, Tier 4**: cleanup.
